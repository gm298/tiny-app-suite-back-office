Operations Manual

Step-by-step guide for Tiny App Suite on tablet (APK). Covers every module from sign-in through settings. Optimized for Tiny Healthy Family Cafe staff and managers.

Tablet first Dark UI v1.0

Reload & Sync

Most data screens show a sync bar at the top. Use it throughout your shift so data stays current and your entries reach the server.

  • Reload — download the latest data from the server.
  • Sync — upload pending changes from this device.
  • Pending count — unsynced items; aim for 0 before end of shift.

Navigation

  • Top bar — Tiny logo (tap → Home), screen title, menu ☰.
  • Menu — sections 1–6 plus Master data, Admin panel, App settings (by role).
  • Large tablet (landscape) — side panel may stay open; use minimize to collapse.
On web: navigation uses the ☰ flyout menu; barcode features are disabled on several screens (see each module).

Sign in with Google

You must sign in before business data appears. Use the Google account that matches your admin invite.

How to sign in (tablet)

  1. Open the app on the tablet.
  2. Tap Continue with Google.
  3. Choose the invited Google account.
  4. Complete sign-in if a browser opens.
  5. Wait until Finishing sign-in… clears.
  6. You should land on Home.

Dashboard

Visual analytics: inventory, production, and waste pie charts. Pick dates and filters; open fullscreen; export CSV/PDF where allowed.

Manager/admin only: Sales and COGS charts on Dashboard; chart PDF export. Staff may export chart CSV where enabled.

COGS calculator

Manager/admin only.

Cost of goods sold for Food, Beverage, or Retail between opening and closing on-hand dates.

  1. Open COGS from Home or menu.
  2. Set Opening and Closing dates.
  3. Select segment tab.
  4. Review the calculation table; Reload if stale.

Purchase order (Kitchen · Bar · Retail)

Same screen for all sections; products and budget differ. Create POs, scan barcodes on tablet, export PDF for suppliers.

On web: barcode scanner disabled — use product search only.

Section & budget

  1. Tap Kitchen, Bar, or Retail (clears unsaved lines).
  2. Kitchen/Bar: review PO budget (monthly cap, today remaining).

Supplier, header, products, save

  1. Choose supplier from search list.
  2. Fill header: Purchase Date, Branch, Required Date, optional notes.
  3. Add lines: scan barcode (tablet) or product search; set qty, discounts, VAT.
  4. Tap Create purchase order; then Sync.
  5. Recent POs: Edit, Export PDF, Share PDF.

Purchase order — full workflow

PO receive

Manager/admin only.

Receive today’s purchase orders into inventory. Edit qty and unit price before posting.

  1. Open PO receive; select section.
  2. Expand a PO sequence.
  3. Select lines; adjust qty/price if needed.
  4. Receive selected or Receive all.
  5. Sync; verify Stock on hand / Stock movement.

Expenses

Manager/admin only.

Record expenses manually or from received POs. PO lines do not add stock again.

  1. Pick section; set contact and chart of accounts.
  2. Add manual lines or import from received PO.
  3. Review staged list; Save; Sync.

Production

Log runs from BOM: form, production log (Planned / Completed today), ingredient recipe table.

  1. Search BOM product; set target amount; Planned or Completed.
  2. Tap + Add production log.
  3. Select run in log; view ingredients; Complete / Delete as needed.
  4. Sync after changes.

Production — step by step

1 / 4

Sales inventory

Manager/admin only. Staff may not load sales data.
  1. Reload sales data.
  2. Filter by date or view all pending.
  3. Select lines; Process selected or Process all.
  4. Sync; check Warning log for unmapped menus.

Stock opname (Kitchen · Bar · Retail)

Physical count per section. On-hand updates only after manager approval.

On web: product search only — no barcode.
  1. Open section opname; Reload if products empty.
  2. Scan or search product; enter counted qty; Add count line.
  3. Submit for approval when complete; Sync regularly.

Stock opname — count workflow

1 / 5

Opname approval

Manager/admin only.
  1. Select section; optional date filter.
  2. Expand session; edit qty if needed.
  3. Approve or Reject; Sync; export CSV/PDF if needed.

Stock on hand

Read-only balances by section. Search to filter products.

Stock movement

Ledger of ins/outs. Filter Kitchen / Bar / Retail / All; search product, type, or reference.

Warning log

Device-local warnings (unmapped sales, negative stock, etc.). Clear after review.

Waste (Kitchen · Bar · Retail)

Record today’s waste per section. Reduces stock on sync.

On web: search + manual qty only.

Master data

Admin only.

Sections: All Product, Purchase Products, BOM, Supplier List, Chart of Accounts, PO Budget.

Admin panel

Admin only.

Invite users, assign roles, view activity log.

App settings

Account (email, role, sign out) and Appearance (Light / Dark / System).

Admin only: data connection details and Reload all data.
